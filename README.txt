CONSIGNES DE PUSH :

1ère règle : TOUJOURS SYNCHRONISER (Bouton Sync. en haut à droite) avant
de travailler sur une classe

2ème règle :  TOUJOURS SYNCHRONISER après avoir fait un commit
-> Sinon le travail ne sera pas actualisé

3ème règle : Ne pas travailler sur une classe si celle-ci est en cours de 
modification
ETAPES DE RÉCUPÉRATION

1. Faire une synchronisation (je me répète mais c'est important)
2. Créez une workspace sur eclipse qui vous sera propre
3. Créez un projet java (qui vous permettra de déposer vos classes)
4. Allez dans : GitHub > FuturConnected_2.0 > src
5. Copiez la classe dans laquelle vous souhaitez travailler dans le dossier src
de votre workspace (Ctrl+c / Ctrl +V, au cas où ...)
6. Faites un F5 sur Eclipse
7. La classe est maintenant présente sur votre projet

ETAPES DE DEPOT

1. SYNCHRONISER
2. Sauvegardez la classe que vous souhaitez déposer
3. Copiez la classe depuis le dossier de votre workspace (workspace > src > classe.java)
4. Collez là dans GitHub > FuturConnected_2.0 > src
5. Sélectionnez l'onglet "Uncommited changes" (avec le nombre de modifications
apportées si tout s'est bien passé)
6. Nommez votre commit (la modification apportée en général) et mettez y une description
si c'est trop complex
7. Cliquez sur "commit"
8. Synchronisez

Normalement votre commit est passé ainsi que la modification que vous avez apporté
(ajout de classe, modification de classe, ajout d'un fichier, ...)
